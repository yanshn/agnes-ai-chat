## 一、修复列表

### 1. 重新生成不产生重复消息块

**根因**：`regenerateFromUser` chat 分支调用 `sendChatMessage`，后者会新增 user+AI 两条消息，和已有 user 消息叠加形成 [旧user, 新user, 新AI] 三块。

**修复**：参考图片/视频模式的 regenerate 机制 ——
- 不删除旧 AI 消息，就地重置（`text=''`, `streaming=true`, 新 `streamId`）
- 局部更新 DOM 行（`querySelector([data-msg-idx])` → 更新 `.bubble.ai`）
- 提取 `streamChatResponse()` 独立函数，供正常发送和再生共用
- `sendChatMessage` 精简为：添加 user → 添加 ai 占位 → 调 `streamChatResponse`

### 2. 输入框提交后立即清空

**根因**：`sendMessage()` chat 分支把 `$promptInput.value=''` 写在 `await sendChatMessage(prompt)` 之后，要等 SSE 流式完成才清空。

**修复**：把清空四行移到 `await` 之前。

### 3. AI 消息块增加复制按钮（排除 thinking）

**按钮位置**：AI 消息底部元信息栏
**实现**：`copyAiMessage(btn)` — 从 `.bubble.ai` 找 `.chat-bubble-ai`，克隆后移除 `.thinking-block`，取 `textContent` 写入剪贴板。按钮反馈 "✅ 已复制" 1.5s。

### 4. 对话消息收集规则

`buildChatMessages(prompt, stopIdx)`：

| 规则 | 实现 |
|------|------|
| 只收集文字对话消息 | 检查 `msg.apiBodies._endpoint === CONFIG.chatEndpoint` |
| 按顺序截断 | `stopIdx` 参数：正常=全部，再生=`userIdx+1` |
| 最多 30 条 | `messages.slice(-30)`，system prompt 不计入 |
| 排除空 AI | `msg.role === 'ai' && msg.text` 才收集 |

`buildRequestBody` 新增 `stopIdx` 参数透传。`regenerateFromUser` 中临时 splice 出旧 AI → `buildRequestBody('', [], userIdx+1)` → splice 恢复。

### 5. 三条路线渲染互不交叉

- `buildBubbleContent` 保持 `getChatMode()` 全局判断 → 生图/生视频渲染逻辑完全不动
- 唯一交集：对话模式下产生的生图消息 → `isImageOrVideoMsg` 检查 → 按卡片渲染
- 生文字→气泡，生图→卡片，生视频→卡片，互不污染
